# 输出格式规范

本 Skill 采用**双输出模式**:先输出审查摘要报告,再以报告为核对清单逐条生成审阅稿/批注稿。

---

## 〇、审查模式(审查前必须确认) ⭐

在开始审查前,**必须先向用户确认审查模式**:

| 模式 | 名称 | 正文处理 | 批注处理 | 适用场景 |
|------|------|---------|---------|---------|
| **模式A** | 修改+批注 | 🔴🟡条款用审阅模式修改(删除原文+插入修改文本),💼条款不修改 | 🔴🟡条款添加修改理由批注,💼条款添加风险提示批注 | 用户确认可以修改合同 |
| **模式B** | 仅批注 | 正文**不做任何修改** | 所有🔴🟡💼条款均添加批注(风险等级+建议) | 用户要求仅做风险提示 |

⚠️ **默认模式**:若用户未明确选择,默认按**模式B**(仅批注)执行。

---

## 一、输出B:审查摘要报告(先输出) ⭐

审查摘要报告必须**先于审阅稿输出**,作为后续生成审阅稿的**核对清单**。

- **合作主题**:______(一句话概括)

```

| 序号 | 条款位置 | 风险等级 | 处理方式 | 修改/批注内容 | 法律依据 | 审阅稿已体现✅/❌ |
|------|---------|---------|---------|-------------|---------|----------------|
| 1 | 第X条 | 🔴 | 修改+批注(模式A)/仅批注(模式B) | [修改内容] | [依据] | ❌→✅ |
| 2 | 第X条 | 🟡 | 修改+批注(模式A)/仅批注(模式B) | [修改内容] | [依据] | ❌→✅ |

| 4 | 第X条 | 💼 | 仅批注 | [批注内容] | - | ❌→✅ |

⚠️ **核对规则**:生成审阅稿后,必须回查此表,确认所有🔴🟡💼行的"审阅稿已体现"列均为✅。若有❌,必须补充对应修改/批注。

## 二、输出A:Word 审阅稿/批注稿(后输出)

以审查摘要报告中的"修改建议汇总"为核对清单,**逐条**在原合同上操作,确保不遗漏。

### 2.1 基本规则

| 项目 | 规范 |
|------|------|
| 修改方式 | **审阅模式(修订模式/Track Changes)** |
| 审阅人名称 | **legal assistant** |
| 输出格式 | **.docx**(Word 格式) |
| 文件命名 | 模式A:`[原合同名]_审查稿_YYYYMMDD.docx` / 模式B:`[原合同名]_批注稿_YYYYMMDD.docx` |

### 2.2 模式A:修改+批注规则

#### 🔴 高风险条款 → 直接修改 + 批注说明

- 在正文中用审阅模式**删除原文、插入修改后文本**
- 同时在旁边**添加批注**,说明:
  - 风险等级:🔴高风险
  - 修改理由
  - 法律依据(来自依据池)

**批注格式**:
```
【🔴高风险】[修改理由]。法律依据:《XXX》第X条。
```

**示例**:
```
【🔴高风险】原条款将乙方责任扩大至"任何第三方索赔",无赔偿上限,对乙方极不公平。已添加赔偿上限条款。法律依据:《民法典》第585条第2款。
```

#### 🟡 中风险条款 → 直接修改 + 批注说明

- 在正文中用审阅模式**删除原文、插入修改后文本**
- 同时在旁边**添加批注**,说明:
  - 风险等级:🟡中风险
  - 修改理由
  - 法律依据

**批注格式**:
```
【🟡中风险】[修改理由]。法律依据:《XXX》第X条。
```

#### 🟢 低风险条款 → 不修改

- **不在正文中修改**
- **不添加批注**
- 仅在审查摘要报告(输出B)中列出

#### 💼 商务条款(仅当事人可确认) → 只批注不修改

以下类型条款属于商务条款,**不在正文中修改**,只在旁边**添加批注**提示风险:

| 商务条款类型 | 批注内容要求 |
|-------------|------------|
| 合同金额/费用/对价 | 提示是否含税、付款节点是否可执行等风险 |
| 服务内容/范围 | 提示是否与附件一致、边界是否清晰等风险 |
| 合作期限/数量 | 提示是否明确、续约条件等风险 |
| 业绩承诺/KPI | 提示是否可实现、违约后果等风险 |

**批注格式**:
```
【💼商务条款】[风险提示]。此条款需合同当事人确认,未做修改。
```

#### 条款缺失时的处理

当审查发现应包含但合同中缺失的条款:

- 在合同**适当位置**(通常为该条款应在的章节末尾)用审阅模式**插入新条款文本**
- 旁边添加批注:
```
【🔴/🟡新增条款】[新增理由]。建议补充此条款以保护乙方权益。法律依据:《XXX》第X条。
```

### 2.3 模式B:仅批注规则

模式B下,正文**不做任何修改**,所有风险提示均通过**批注**方式添加:

#### 🔴🟡 风险条款 → 仅批注

**批注格式**:
```
【🔴/🟡风险提示】[问题分析]。建议修改为:[修改建议]。法律依据:《XXX》第X条。此为风险提示,未修改正文。
```

#### 💼 商务条款 → 仅批注

**批注格式**:
```
【💼商务条款】[风险提示]。此条款需合同当事人确认,未做修改。
```

```

### 2.5 WPS COM 技术要点 ⭐

使用 WPS COM 自动化生成审阅稿时,必须遵循以下技术规范:

#### 关键发现：Find/Replace 不产生 Track Changes

WPS COM 的 `Range.Find.Execute(Replace=2)` 即使在 `TrackRevisions = True` 状态下,**也不会产生修订记录**。必须使用 **Delete + TypeText** 模拟用户输入才能正确生成 Track Changes。

#### 正确的技术流程（模式A）

```
Phase 1: 修改+批注（逐条执行）
  对每条修改:
  1. Range.Find.Execute(FindText=原文) → 找到目标范围
  2. Selection.SetRange(rng.Start, rng.End) → 选中
  3. doc.Comments.Add(Selection.Range, 批注内容) → 添加批注
  4. Selection.Delete() → 删除原文（产生修订删除标记）
  5. Selection.TypeText(新文) → 键入新文（产生修订插入标记）
     注意：多行文本用 TypeParagraph 处理换行

Phase 2: 商务条款批注
  对每条商务条款:
  1. Range.Find.Execute(FindText=条款片段) → 找到
  2. doc.Comments.Add(rng, 批注内容) → 添加批注
```

#### 错误做法（会导致修订不显示）

| 错误做法 | 后果 |
|---------|------|
| ❌ 先添加所有批注,再执行 Find/Replace | Find/Replace 不产生修订记录,批注正常但修改不可见 |
| ❌ 使用 Range.Find.Execute(Replace=2) 替代 Delete+TypeText | 不产生修订记录,文本被直接替换无痕迹 |
| ❌ 使用 Selection.Find 连续搜索（不重置位置） | 后续搜索从前一次修改位置开始,可能找不到目标 |

#### 搜索技巧

- **使用 Range.Find 而非 Selection.Find**: Range.Find 从文档开头搜索,不受当前光标位置影响
- **长文本搜索失败时**: 逐步缩短搜索片段重试（30字→20字→15字→10字）
- **换行符处理**: 替换文本中的 `\n` 需用 `TypeParagraph()` 替代,不能用 `\n` 直接传入 TypeText

#### WPS COM 初始化

```python
wps = win32com.client.Dispatch('Kwps.Application')
wps.UserName = 'legal assistant'  # 审阅人名称
doc = wps.Documents.Open(src)
doc.TrackRevisions = True  # 必须在修改前开启
```

### 2.6 审阅稿核对机制 ⭐

生成审阅稿后,必须执行以下核对:

1. **打开审查摘要报告的"修改建议汇总"表**
2. **逐条核对**:每一条🔴🟡💼的修改/批注是否已在审阅稿中体现
3. **发现遗漏**:立即补充对应修改/批注,不可跳过
4. **全部核对通过后**:在自检声明中确认"审阅稿核对通过"
